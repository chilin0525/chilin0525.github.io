from users.models import User
from tools.R import R

def logincheck(*methods):

            token = request.META.get("HTTP_AUTHORIZATION")
            if request.method not in methods:
                return func(request,*args,**kwargs)
            if not token:
                return R.badRequest("no token")

高階函數 Link to heading

在理解 decorator 的先備知識為理解高階函數，高階函數也是函數的一種，泛指接受函數為輸入(argument)或是輸出的函數，Curry function 就算是高階函數的一種，如果你寫過 React 應該了解 Curry function 的強大處。下面範例中試圖要在 hello 函數回傳 this is hello 前加上 prefix，由 add_prefix 負責加上 prefix，並且接受函數作為輸入，再回傳加上 prefix 的函數，由 hello_w_prefix 接收加上 prefix 的 hello，因此執行 hello_w_prefix 時將會得到加上 prefix 的 hello：

def hello():
    """I am hello
    """
    print("this is hello")
    return "this is hello"


def add_prefix(hello):
    def add_prefix_to_hello():
            """I am add_prefix
            """
            print("adding prefix")
            return "this is prefix before" +  hello()
    return add_prefix_to_hello


hello_w_prefix = add_prefix(hello)
print('-'*10)
print(hello_w_prefix.__name__)
print(hello_w_prefix.__doc__)
print('-'*10)
print(hello_w_prefix())
print('-'*10)

結果:

----------
add_prefix_to_hello
I am add_prefix

----------
adding prefix
this is hello
this is prefix beforethis is hello
----------

另外值得注意的是 hello_w_prefix 這個 function name 將不是 hello，而是 add_prefix_to_hello，不僅是函數名稱連 docstring 都被替換掉了，然而我們通常只希望 add prefix 這個過程是暫時的而非永久替換原先函數。

Tip
add prefix 就是個 decorator，因此我們可以理解為 decorator 是個高階函數，接受了某個函數並裝飾/修改了函數本身後再回傳包裝後的函數

因此如果試著改寫上面的程式碼，以語法糖形式呈現，但仍然沒有解決 function name 被覆蓋問題:

def add_prefix(hello):
        def add_prefix_to_hello():
                """I am add_prefix
                """
                print("adding prefix")
                return "this is prefix before" +  hello()
        return add_prefix_to_hello


@add_prefix
def hello():
        """I am hello
        """
        print("this is hello")
        return "this is hello"

print('*'*10)
print(hello())
print('*'*10)
print(hello.__name__)

**********
adding prefix
this is hello
this is prefix beforethis is hello
**********
add_prefix_to_hello

Syntax Link to heading

  • 語法
    • 在 PEP3181 中以下列範例說明 decorator syntax: 
      @dec2
@dec1
def func(arg1, arg2, ...):
    pass

# equal to following
def func(arg1, arg2, ...):
    pass
func = dec2(dec1(func))
  • 定義多個 decorator，最近的優先呼叫，如 dec1
  • decorator 可帶入參數，上面例子中 prefix 被寫死了，如果我們希望客製化 prefix，其實只要自行定義 function 並將 prefix 作為參數，return decorator 即可！
    • 醜陋版本: 
      def define_prefix(prefix):
    def add_prefix(hello):
        def add_prefix_to_hello():
            """I am add_prefix
            """
            print("adding prefix")
            return f"this is {prefix} before" +  hello()
        return add_prefix_to_hello
    return add_prefix


add_a_prefix = define_prefix('a')
@add_a_prefix
def hello():
    """I am hello
    """
    print("this is hello")
    return "this is hello"


print('*'*10)

  • 消除 function name 被覆蓋的副作用: 
    from functools import wraps


def define_prefix(prefix):
    def add_prefix(func):
        @wraps(func) # 注意這裡
        def add_prefix_to_hello():
                """I am add_prefix
                """
                print("adding prefix")
                return f"this is {prefix} before" +  func()
        return add_prefix_to_hello
    return add_prefix


@define_prefix('a')
def hello():
    """I am hello
    """
    print("this is hello")
    return "this is hello"


print(hello.__name__)
print(hello())

# hello
# adding prefix
# this is hello
# this is a beforethis is hello

Sumamry Link to heading

總結一下這篇文章:

  • decorator 為高階函數，修飾作為參數輸入的函數並回傳裝飾後的函數
  • 多個 decorator 由最近的先呼叫執行
  • 如 decorator 有輸入參數需求，可再寫一高階函數接受參數並回傳 decorator
  • 使用 functools 的 wraps 消除函數名稱被覆蓋的副作用
